# 编辑器右键菜单设计

## 背景

当前编辑器正文区域使用 `WKWebView` 承载自定义 Markdown 编辑器。正文区右键时，系统直接弹出 WebKit 默认网页上下文菜单，包含“查询”“翻译”“用 Bing 搜索”“分享”等浏览器/系统文本服务项，不符合当前项目的 Markdown 编辑场景。

## 目标

1. 保留原生编辑项中的基础能力：撤销、重做、剪切、复制、粘贴、全选。
2. 在同一菜单中加入当前项目常用 Markdown 命令。
3. 不重做编辑器视觉，不引入前端自绘菜单。
4. 继续复用现有 `EditorCommand` / `runCommand` 执行链。

## 方案

### 方案选择

采用“原生层接管 `WKWebView` 右键菜单，在系统菜单基础上裁剪并插入 Markdown 命令”的方案。

不采用纯前端菜单，原因是：

- 会丢失原生菜单手感和 responder 链。
- 需要额外处理复制、粘贴、关闭菜单、键盘可达性。
- 对当前项目来说，已有命令体系可以直接复用，没有必要再造一层。

### 菜单结构

正文区右键菜单分为三段：

1. 原生编辑项：撤销、重做、剪切、复制、粘贴、全选。
2. 行内格式：粗体、斜体、插入链接、行内代码。
3. 块级结构：标题 1、标题 2、标题 3、引用块、无序列表、有序列表、任务列表、代码块。

### 过滤规则

从 `WKWebView` 默认菜单中移除以下明显属于网页/系统文本服务但不适合当前编辑器的项：

- 查询…
- 翻译…
- 用 Bing 搜索
- 拼写和语法
- 替换
- 转换
- 字体
- 语音
- 段落方向
- 分享…
- 自动填充
- 拍照
- 扫描文稿
- 添加速绘

基础编辑项保留，不重新实现。

### 命令执行

Markdown 菜单项不直接操作 DOM，也不新增前端命令入口。它们统一调用原生侧已有的 `EditorCommand`，再走 `EditorWebView.Controller.runCommand(_:)` 发送到网页编辑器。

这样菜单栏命令、命令面板、右键菜单会共用同一条执行链。

## 边界

这轮不做：

- 复杂上下文菜单，例如不同选区状态显示不同项。
- 表格专用右键重构。
- 链接、图片、代码块等更细粒度的上下文判断。
- 前端自绘悬浮菜单。

## 验证

1. 手工在正文区右键，确认默认网页菜单被替换。
2. 验证“剪切/复制/粘贴/全选”仍可用。
3. 验证“粗体/斜体/链接/标题/列表/代码块”能作用于当前文档。
4. 跑 `xcodebuild build -scheme Markdown -destination 'platform=macOS'`。
5. 跑与编辑器和文档控制器相关的定向测试。
