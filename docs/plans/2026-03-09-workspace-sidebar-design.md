# Workspace Sidebar Design

**日期：** 2026-03-09  
**状态：** 已确认，待实施

## 目标

把当前只能“查看并点击打开文件”的侧边栏文件树，改造成真正可用的 Markdown 项目树：

- 文件夹默认支持折叠 / 展开
- 展开状态不再每次刷新都全开
- 文件、文件夹和空白区域都具备合理的右键菜单
- 支持创建文件、创建文件夹、重命名、删除、在 Finder 中显示

## 当前问题

当前实现主要由 [`ContentView.swift`](/Users/math173sr/Documents/GitHub项目/Markdown/Markdown/ContentView.swift)、[`EditorDocumentController.swift`](/Users/math173sr/Documents/GitHub项目/Markdown/Markdown/EditorDocumentController.swift) 和 [`EditorWorkspaceTree.swift`](/Users/math173sr/Documents/GitHub项目/Markdown/Markdown/EditorWorkspaceTree.swift) 组成。

现状问题有三个：

1. 文件树默认全展开  
   `refreshWorkspace()` 和 `openFolder(at:)` 会直接把所有文件夹 id 填进 `expandedFolderIDs`，导致工作区一打开就是全部展开。

2. 文件树只有只读交互  
   目前文件夹只支持点击折叠，文件只支持点击打开，没有创建、删除、重命名等文件系统能力。

3. 没有右键菜单  
   侧边栏没有上下文菜单，因此无法像一个标准项目树那样完成日常文件组织。

## 设计选择

采用“保留现有自定义树视图，补齐项目树能力”的方案。

原因：

- 当前项目已经有成熟的侧边栏视觉和自定义缩进、选中态，不需要为了一个文件树交互把整套 UI 换成 `NSOutlineView`。
- 这轮目标是把项目树能力做实，不是重写侧边栏技术栈。
- 后续如果要补拖拽、多选、文件监听，再决定是否升级到底层 AppKit 控件。

## 交互设计

### 默认折叠 / 展开

- 文件夹行始终显示左侧箭头
- 点击箭头折叠 / 展开
- 点击文件夹名称也折叠 / 展开
- 文件行保持点击即打开文档

### 展开状态策略

- 打开一个新文件夹时，只默认展开根层目录
- 手动展开 / 收起状态由 `expandedFolderIDs` 持久维护
- 工作区刷新时尽量保留已有展开状态
- 新创建的文件夹在创建后自动展开

### 右键菜单

#### 文件

- 新建同级文件
- 重命名
- 删除
- 在 Finder 中显示

#### 文件夹

- 新建文件
- 新建文件夹
- 重命名
- 删除
- 折叠 / 展开
- 在 Finder 中显示

#### 空白区域

- 新建文件
- 新建文件夹
- 在 Finder 中显示当前目录

## 文件系统行为

### 创建文件

- 默认创建为 `.md`
- 如果没有后缀，自动补成 `.md`
- 默认内容为空
- 创建成功后刷新工作区并自动打开该文件

### 创建文件夹

- 在目标目录下创建新文件夹
- 创建成功后刷新工作区并自动展开该目录

### 重命名

- 文件与文件夹都支持重命名
- 使用原生弹窗输入新名称
- 文件重命名时保持 `.md` 兼容行为
- 如果当前打开的是被重命名文件，需要同步更新对应标签页 URL 和标题

### 删除

- 删除前弹确认框
- 先按系统垃圾桶语义处理，如果当前实现成本过高则先使用直接删除
- 如果删除的是当前打开文件，需要关闭对应标签页或回退到其它打开标签

### 在 Finder 中显示

- 统一使用 `NSWorkspace.shared.activateFileViewerSelecting`

## 范围边界

本次不做：

- 拖拽移动文件或文件夹
- 多选 / 批量操作
- 文件系统实时监听
- 内联重命名
- 空文件夹常驻显示（当前树仍然基于 Markdown 文件列表推导）

## 验收标准

1. 侧边栏文件夹默认支持点击折叠 / 展开
2. 打开工作区后不再全展开所有层级
3. 文件、文件夹、空白区域都有可用右键菜单
4. 可以从侧边栏直接创建文件、创建文件夹、重命名、删除
5. 工作区刷新后用户的展开状态不会被粗暴重置
